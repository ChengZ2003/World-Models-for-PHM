# Maintenance Guide

## Routine Review

1. Validate every structured-data pull request.
2. Require primary or official sources for metadata changes.
3. Regenerate derived tables and review diffs.
4. Check links and last-checked dates without silently treating redirects as verification.
5. Record classification disagreements and failed reproductions.

## Release Hygiene

Update `CHANGELOG.md`, version fields, status tables, badges, generated files, and checklists together. Do not mark a roadmap item complete because a placeholder file exists. Review stale `TODO` values and never replace them by inference alone.

## Taxonomy Changes

Changes to category meaning should include motivation, examples based on verified records, migration impact on CSV values, and alignment with the survey text.
