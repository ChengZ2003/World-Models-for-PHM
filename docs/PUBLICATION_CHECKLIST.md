# Public Release Checklist

## Repository Content

- [ ] Repository name checked
- [ ] Description checked
- [ ] README reviewed
- [ ] Scope reviewed
- [ ] At least one manually reviewed taxonomy example
- [ ] Generated files match CSV
- [ ] No duplicated manually maintained paper tables
- [ ] No broken badges
- [ ] Current Stage is visible
- [ ] Benchmark status is not overstated
- [ ] No fabricated papers
- [ ] No fabricated repositories
- [ ] No private information
- [ ] No local file paths
- [ ] No fake author or contact information
- [ ] Author placeholders reviewed and replaced or explicitly approved
- [ ] Maintainer conduct-reporting contact confirmed
- [ ] `CITATION.cff` placeholders reviewed
- [ ] License included
- [ ] Contribution guide included
- [ ] Scripts pass
- [ ] GitHub Actions passes
- [ ] Relative links pass
- [ ] Source-of-truth policy is documented
- [ ] TODOs reviewed
- [ ] Example rows clearly labeled
- [ ] Example records remain unverified
- [ ] No claim of completed benchmark
- [ ] No claim of novelty without evidence
- [ ] Repository visibility decision confirmed

## Minimum Useful-Content Target

These targets are recommended for a useful initial public release; they are not hard requirements enforced by validation scripts.

- [ ] 10–20 manually verified papers
- [ ] 6–10 verified related repositories
- [ ] 3–5 verified datasets
- [ ] 2–3 complete paper notes
- [ ] At least one worked classification example
- [ ] README reflects actual content counts

> The repository may technically be public before these targets are reached, but meeting them is recommended to avoid publishing an empty template-only project.
